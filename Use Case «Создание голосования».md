## Use Case: Создание голосования

### Название
Create a poll (Создание голосования)

### Роли (Акторы)
- **Пользователь (Студент / Сотрудник МИФИ)** — инициирует процесс создания опроса, заполняет параметры и публикует его.
- **Система «Атомный выбор»** — проверяет право на создание, валидирует данные, сохраняет опрос, обеспечивает отображение для голосующих.
- **База данных пользователей МИФИ** — хранит актуальные данные о студентах и сотрудниках для верификации.

### Предусловия
1. Пользователь авторизован в системе «Атомный выбор» через учётную запись МИФИ.
2. Пользователь относится к целевой аудитории (студент или сотрудник МИФИ).
3. У пользователя есть право на создание опросов (все авторизованные пользователи).
4. Система доступна и функционирует в штатном режиме.

### Основной поток

1. **Пользователь переходит в раздел «Мои опросы» или нажимает кнопку «Создать опрос» на главной странице.**

2. **Система отображает форму создания опроса с полями:**
   - Название опроса (обязательное)
   - Описание (необязательное)
   - Вопрос (или несколько вопросов)
   - Тип ответа (одиночный выбор / множественный выбор)
   - Варианты ответов (минимум 2)
   - Дата и время начала
   - Дата и время окончания
   - Настройки доступа (все пользователи / только определённые группы)

3. **Пользователь заполняет все обязательные поля, при необходимости добавляет дополнительные вопросы и варианты ответов.**

4. **Пользователь нажимает кнопку «Опубликовать» или «Сохранить как черновик».**

5. **Система проверяет корректность заполнения:**
   - Название опроса не пустое.
   - Минимум один вопрос.
   - Каждый вопрос имеет минимум два варианта ответа.
   - Дата окончания позже даты начала.
   - Дата начала не раньше текущего момента.

6. **Система сохраняет опрос в базе данных с уникальным идентификатором.**

7. **Система присваивает опросу статус «Активен» (при публикации) или «Черновик» (при сохранении).**

8. **Система отображает сообщение: «Опрос успешно создан!» и перенаправляет пользователя на страницу управления опросом.**
### Расширения

**2а. Пользователь не подтверждён как студент или сотрудник МИФИ**
- 2а.1. Система определяет, что учётная запись пользователя не прошла верификацию.
- 2а.2. Система отображает сообщение: «Только студенты и сотрудники МИФИ могут создавать опросы».
- 2а.3. Система предлагает обратиться в службу поддержки или завершить авторизацию через портал МИФИ.
- 2а.4. Use case завершается.

**5а. Некорректно заполнены обязательные поля**
- 5а.1. Система проверяет форму на наличие ошибок.
- 5а.2. Система подсвечивает поля с ошибками (пустое название, недостаточно вариантов ответа и т.д.).
- 5а.3. Система отображает сообщение: «Пожалуйста, заполните все обязательные поля корректно».
- 5а.4. Пользователь исправляет ошибки и повторяет попытку публикации.

**5б. Дата окончания раньше даты начала**
- 5б.1. Система проверяет корректность временных интервалов.
- 5б.2. Система выводит сообщение: «Дата окончания не может быть раньше даты начала».
- 5б.3. Пользователь корректирует даты и повторяет попытку.

**5в. Дата начала раньше текущего времени**
- 5в.1. Система проверяет дату начала относительно текущего момента.
- 5в.2. Если дата начала в прошлом, система предлагает установить текущее время или будущее.
- 5в.3. Пользователь корректирует дату.

**6а. Ошибка сохранения опроса**
- 6а.1. В момент сохранения происходит сбой (проблема с БД, потеря соединения).
- 6а.2. Система отменяет операцию.
- 6а.3. Система уведомляет пользователя: «Произошла ошибка. Пожалуйста, попробуйте снова».
- 6а.4. Данные формы сохраняются в локальном хранилище или предлагается восстановить черновик.
- 6а.5. Пользователь повторяет попытку сохранения.

**7а. Пользователь создаёт опрос с множественными вопросами**
- 7а.1. Система позволяет добавлять/удалять вопросы динамически.
- 7а.2. Для каждого вопроса настраивается отдельный тип ответа и варианты.
- 7а.3. Поток сохраняется без изменений.

### Постусловия

1. Опрос сохранён в системе с уникальным идентификатором и статусом («Активен» или «Черновик»).
2. Создатель опроса связан с опросом для возможности редактирования и просмотра статистики.
3. При статусе «Активен» опрос становится доступен для голосования целевой аудитории.
4. Система фиксирует дату и время создания опроса.
